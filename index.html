源稿与规范

源稿与维护层

以后如果你要继续更新、补证据或重做某个 deck，优先回到这里。

sources/md/

  • 保留每份“主报告”的 Markdown 源稿
  • 避免只有 HTML、无法继续维护

sources/notes/

  • 保留量化备忘录、可比案例笔记等证据底稿
  • 不在对外入口抢位置，但未来可回溯

DESIGN_RULES.md

  • 统一后续 deck 的视觉语言、信息密度与交互边界
  • 让未来新增报告继续像同一个网站，而不是新起一套壳

建议未来直接以这个目录为 repo 根目录，新建仓库后开启 GitHub Pages（Deploy from branch / root）即可。
