附录 / 可比案例

“官网做独立 WebApp + 网页付费点” 在欧美不是试验，而是成熟做法。

这份附录用 Calm、Insight Timer、BetterSleep、Spotify / Netflix 等案例，证明网页支付、跨端权益和官网独占 SKU 在欧美已经被长期验证。

Core Pattern

网页支付

Stripe / 信用卡 / 官网订阅页，承担真正的结算职责。

跨端权益

在网页付费，在 App 消费，是典型成熟路径。

官网独占 SKU

终身、礼品卡、学生计划、企业福利更适合放官网。

Case Evidence

Calm

  • 官网独占终身会员
  • 支持网页支付与跨端使用
  • 说明高客单 SKU 放 Web 是成熟动作

Insight Timer

  • 明确提供 Web Application
  • 支持网站订阅 MemberPlus
  • 社区规模达到 3000 万成员级别

BetterSleep

  • 官网购买订阅后可跨 iOS / Android 使用
  • 证明助眠类产品已经验证网页支付
  • 下载量达到 6500 万级别

Spotify / Netflix

  • 把 App 端当消费端，把网页端当支付端
  • 减少平台限制和渠道抽成
  • 是产业级背书，而不只是 niche 案例
Implication

对 SleepAI 的可迁移结论

  • Web 应成为跨端统一账号体系和结算端
  • 高客单 SKU 与优惠体系优先放官网
  • 移动端 App 负责消费体验，网页端负责增长与订单

为什么这很重要

  • 它不是“绕过应用商店”的小技巧，而是成熟业务结构
  • 对音频 / 助眠 / 订阅内容产品尤其成立
  • 如果不做网页支付，长期会损失定价权和促销灵活度
一句话：Web 付费点在欧美早就不是冒险，而是订阅业务的标准基础设施。
